## Prerequisites

Before you start, make sure you have:

- OSGUI installed — see the [installation guide](/installation) if you haven't done this yet
- `yt-dlp` available on your system `PATH`
- `ffmpeg` (including `ffplay` and `ffprobe`) available on your system `PATH`

## Download your first video

<Steps>
  <Step title="Launch OSGUI">
    Open OSGUI from your application launcher, Start menu, or Applications folder. The app performs a startup check for `yt-dlp` and `ffmpeg`. If either binary is missing, you will see a warning toast — resolve that first by following [Dependencies](/configuration/dependencies).
  </Step>
  <Step title="Paste a video URL">
    Click the URL input field at the top of the main view and paste the link to the video you want to download. You can type the URL manually or use the clipboard paste button next to the field to paste from your clipboard automatically.

    <Tip>
      OSGUI supports YouTube, Facebook, X (Twitter), and 1,000+ other sites. Any URL that `yt-dlp` supports will work here.
    </Tip>
  </Step>
  <Step title="Click Search">
    Click the **Search** button. OSGUI runs `yt-dlp --dump-json` against the URL to fetch video metadata — title, available formats, and stream information — without starting a download. This usually completes in a few seconds.
  </Step>
  <Step title="Select a format">
    Once metadata loads, two dropdowns appear: one for the **video stream** and one for the **audio stream**. Select your preferred quality from each dropdown.

    - Selecting only a video stream downloads that stream alone.
    - Selecting both a video stream and an audio stream combines them into a single file using `ffmpeg` (format string: `videoId+audioId`).
    - If you skip format selection, OSGUI downloads the best available format automatically.
  </Step>
  <Step title="Click Download">
    Click the **Download** button. OSGUI adds the item to the download list and invokes `yt-dlp` with your selected format:

    ```bash
    yt-dlp -f <formatString> -o "<Downloads>/OSGUI/%(title)s<formatString>.%(ext)s" <videoUrl>
    ```

    The download list updates in real time, showing the live status message from `yt-dlp`'s output as the file is fetched and processed.
  </Step>
  <Step title="Find your file">
    When the download completes, the status changes to **Download completed successfully!**. Your file is saved to:

    <Note>
      Files are saved to `~/Downloads/OSGUI/` on all platforms. The file name follows the pattern `<video title><formatString>.<ext>`.
    </Note>

    Open your file manager and navigate to the `OSGUI` folder inside your Downloads directory to find the file.
  </Step>
</Steps>

## Tips

**Pausing a download**

Click the **Pause** button next to an active download in the download list. OSGUI uses a stream-interception architecture to stop the `yt-dlp` process cleanly — no zombie processes are left running in the background. The download entry stays in the list with a **Paused by user** status.

To restart a paused download, click the restart icon on that entry. OSGUI will re-submit the same URL and format, starting the download fresh from the beginning.

<Tip>
  Restarting a paused download begins from scratch. For large files, consider letting the download run to completion rather than pausing midway.
</Tip>

**Checking your yt-dlp version**

OSGUI checks your local `yt-dlp` version at startup and compares it against the latest release. If an update is available, a warning toast appears with both the installed version and the current online version. Keep `yt-dlp` up to date for the best site compatibility — new releases ship roughly every two weeks.

```bash
# Update yt-dlp via pip
python -m pip install -U yt-dlp

# Or download the latest standalone binary from:
# https://github.com/yt-dlp/yt-dlp/releases
```
