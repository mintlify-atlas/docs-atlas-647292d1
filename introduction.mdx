OSGUI is a lightweight, cross-platform desktop application that puts a native GUI on top of `yt-dlp`. Instead of typing commands in a terminal, you paste a URL, pick your format, and click Download — OSGUI handles the rest.

## Who it's for

OSGUI is built for people who want the full power of `yt-dlp` without memorizing flags or managing shell scripts. If you download videos regularly and prefer a graphical interface over the command line, OSGUI is for you.

## Supported sites

OSGUI supports every site that `yt-dlp` supports — that includes YouTube, Facebook, X (Twitter), and more than 1,000 other platforms. If `yt-dlp` can download it, OSGUI can too.

## Architecture

OSGUI is built on three layers that stay strictly separated:

- **UI layer** — React functional components styled with [HeroUI](https://www.heroui.com/). The window is rendered via Tauri's WebView.
- **Logic layer** — [Zustand](https://github.com/pmndrs/zustand) stores divided by domain (`ApplicationStore`, `DownloadStore`, `UserInputVideoStore`). High-frequency I/O updates from `yt-dlp` are decoupled from the React render cycle to keep the UI smooth.
- **Data layer** — A local SQLite database (via `tauri-plugin-sql`) persists download history across sessions. The Rust backend invokes `yt-dlp` and `ffmpeg` as sidecar processes via Tauri's shell plugin.

<Note>
  OSGUI does not bundle `yt-dlp` or `ffmpeg`. You must install them separately and make sure they are available on your system `PATH`. The app shows a warning toast on startup if either binary is missing or outdated.
</Note>

## Requirements overview

| Dependency | Required | Notes |
|---|---|---|
| `yt-dlp` | Yes | Must be named `yt-dlp` (or `yt-dlp.exe` on Windows) on PATH |
| `ffmpeg` | Yes | Required for merging and transcoding |
| `ffplay` | Yes | Bundled with most ffmpeg distributions |
| `ffprobe` | Yes | Bundled with most ffmpeg distributions |

For platform-specific installation instructions, see [Dependencies](/configuration/dependencies).
